# Publishing an update

The app uses Tauri auto-update (`tauri-plugin-updater`). Users receive updates automatically on startup.

## Process

1. Bump the version in all three files (same number):
   - `src-tauri/tauri.conf.json` -> `"version": "1.2.0"`
   - `package.json` -> `"version": "1.2.0"`
   - `src-tauri/Cargo.toml` -> `version = "1.2.0"`

2. Write the release notes in `RELEASE_NOTES.md` (displayed in the in-app update modal):

   ```markdown
   - New feature
   - Bug fix
   ```

   **Always keep it very short** (2-4 short lines max). These notes are shown in the
   small in-app update modal, so anything longer gets cut off or overflows — summarize,
   do not list every change. Put nothing but the notes in this file (the CI copies it
   verbatim into the release body).

   The CI reads this file and creates the release with it as the body — no manual
   `gh release edit` afterwards. To fix notes after the fact, you can still run
   `gh release edit vX.Y.Z --notes "..."`.

3. Commit, tag, push:

   ```bash
   git add -A && git commit -m "release v1.2.0"
   git tag v1.2.0
   git push origin main --tags
   ```

4. The `.github/workflows/build-windows.yml` workflow triggers on `v*.*.*` tags:
   builds Windows -> signs -> creates the GitHub Release with `.msi`, `.exe`,
   their `.sig` files, and `latest.json`.

5. Installed apps detect the new version via
   `https://github.com/SylvainBoudacher/xingxing-debrid/releases/latest/download/latest.json`
   and prompt the user to update.

## How it works

- **Signing**: Tauri signs each binary with the private key
  (`~/.tauri/xingxing.key`, passed to CI via the `TAURI_SIGNING_PRIVATE_KEY` secret).
  The app verifies the signature against the public key (`pubkey` in `tauri.conf.json`)
  before installing. An update not signed by this key is rejected.
- **Manifest**: `latest.json` (generated by `tauri-apps/tauri-action`) lists the version,
  download URLs, and signatures.

## Required GitHub secrets

- `TAURI_SIGNING_PRIVATE_KEY`: contents of `~/.tauri/xingxing.key`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`: password for the key

## IMPORTANT: do not lose the private key

Back up `~/.tauri/xingxing.key` AND its password in a safe place.
Without this key, it is impossible to sign an update accepted by already-installed apps:
you would have to manually redistribute a new build (with a new `pubkey`) to all users.
